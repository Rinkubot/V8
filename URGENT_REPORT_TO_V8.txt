================================================================================
THE VULNERABILITY
================================================================================

LOCATION: src/compiler/wasm-compiler.cc:2458
FUNCTION: BuildDecodeException32BitValue()

BUG: Exception values are encoded with unsigned conversion but decoded with
     signed conversion, causing sign extension corruption for ~50% of values.

BUGGY CODE (line 2458):
    Node* lower = BuildChangeSmiToInt32(  // Uses SAR (signed)
        gasm_->LoadFixedArrayElementSmi(values_array, *index));

SHOULD BE:
    Node* lower = BuildChangeSmiToUint32(  // Uses SHR (unsigned)
        gasm_->LoadFixedArrayElementSmi(values_array, *index));

IMPACT: Values with bit 15 set get sign-extended:
    0x00008000 → 0xFFFF8000 (corrupted!)
    0x0000FFFF → 0xFFFFFFFF (corrupted!)

THE FIX
================================================================================

REQUIRED CHANGES:

1. Add BuildChangeSmiToUint32() function (uses Word32Shr):

    Node* WasmGraphBuilder::BuildChangeSmiToUint32(Node* value) {
      return COMPRESS_POINTERS_BOOL
                 ? gasm_->Word32Shr(value, BuildSmiShiftBitsConstant32())
                 : BuildTruncateIntPtrToInt32(
                       gasm_->WordShr(value, BuildSmiShiftBitsConstant()));
    }

2. Modify BuildDecodeException32BitValue() line 2458:

    Change: BuildChangeSmiToInt32(...)
    To:     BuildChangeSmiToUint32(...)
